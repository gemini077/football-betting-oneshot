# 自动分析与可选大模型叙述

当前版本：`v0.15.0`

默认分析完全由 `scripts/automatic_model_core.py` 的固定公式完成：近期主客场攻防占 60%，多公司市场校准占 40%，随后生成 λ、胜平负、总进球、BTTS、比分矩阵、逐玩法公平价、EV 与最低可接受赔率。默认流程不调用 DeepSeek，不消耗大模型 token。

## 用户操作

1. 在主页未分析赛事中点击“分析这场”。
2. 页面打开已预填比赛编号、业务日和对阵的 GitHub Issue。
3. 仓库所有者提交后，GitHub Actions 抓取数据、运行确定性模型、生成报告并更新 GitHub Pages。

## 运行策略

- 默认：`FBOS_USE_LLM=0`，固定公式直接生成完整报告。
- 可选叙述：只有手动设置 `FBOS_USE_LLM=1` 且配置 `DEEPSEEK_API_KEY` 时，才调用精简叙述层；概率、方向、价格门槛仍由固定公式覆盖，模型不得改数。
- 临盘：每场只在 `T-6H` 与 `T-90M` 两个检查点刷新一次；没有到达新检查点时，任务不安装依赖、不测试、不部署。
- 赛后：按预计结束时间触发一次，未取得最终赛果时最多执行一次有界重试，然后生成全面复盘。

## 报告口径

- 比分表同时显示概率与公平赔率；具体比分只有完全相同才算命中。
- 欧赔、亚盘、大小球分别显示当前价、模型概率、点估计 EV 与含 8% 安全边际的最低价格。
- 模型含市场校准，因此价格表是复核工具，不宣称独立套利优势。
- 真实赔率轨迹必须至少有 3 个独立快照；不足时明确显示为快照，不伪造趋势图。

## 安全边界

- 自动输出不会建立真实注单、不会锁单、不会修改账户余额。
- 只有用户明确说“锁单/已下单”，真实投注层状态才允许变化。
- 模拟投注最低 2.00 元、步进 0.01 元；没有真实赛前成交价只观察，不计算模拟盈亏。
