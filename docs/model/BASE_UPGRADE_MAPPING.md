## 1. 融合结论

当前模型是 `huqiwengithub/football-odds-analyst` 基础架构与本项目 0—25 流程的融合版本。

在没有明确废弃记录时，基础模块一律继续生效。现有文件中没有发现任何基础模块被明确标记为废弃，因此不得因为升级流程增加了新步骤而静默跳过基础计算。

## 2. 规则优先级

1. `05_RUNTIME_STATE.json`：账户、余额、暴露、锁单状态。
2. `04_CORRECTIONS_模型修正池.md`：已经验证并明确生效的修正规则。
3. `01_MODEL_CORE_赛前流程.md`：升级后的 0—25 分析流程。
4. 基础架构 v3.10.8 主流程及其知识库、投注 SOP。

新规则只有在明确写出“替代哪条旧公式、替代原因、开始生效时间”时才能覆盖旧规则；其余情况均为累加关系。

## 4. 当前明确新增的升级模块

- 正确比分基础组、邻近污染组、右尾/冷门组三组拆分。
- Poisson + Dixon-Coles + 8×8 比分矩阵。
- 总进球桶与 BTTS 独立判断。
- 唯一主维度、唯一首推比分和最大错点。
- 抓取价初算 EV、用户渠道价复算 EV 的两阶段价格审核。
- 账户余额驱动的本金修复期仓位。
- 必须由用户明确说“锁单/已下单”才能改变注单状态。

这些模块是扩展或明确覆盖投注执行参数，不构成删除 MBI、6D、Shin、陷阱扫描的证据。

## 5. 当前数据与实现缺口

| 缺口 | 影响 | 处理规则 |
|---|---|---|
| 欧赔变动历史尚不足24小时 | SCS已能使用多帧序列，但噪声基线仍不完整 | 标记“降级”，不得冒充正式SCS |
| 缺联赛历史中位DRI库 | 只能计算原始离散，不能联赛校准 | 输出原始值与“未校准”标记 |
| 必发近期交易不足4小时基线 | 量价交叉验证不完整 | 交易所模块降权 |
| `shuju`结构化结果可能为空 | ELO、积分、形态、交锋不能自动闭环 | 联网补齐并明确数据来源 |
| 公司层级仅确认13/30家 | 三层共识权重不完整 | 继续识别SBO/IBC、18bet及其余公司cid |
| 知识库“49条”计数冲突 | KB-2的15条 + KB-3的28条 + KB-10的4条仅为47条 | 保留2条上游未定义缺口，不自行编造 |
| 6D、陷阱仅完成可执行骨架 | D1基本面、联赛DRI校准和24条陷阱仍缺可靠字段 | 报告显示计算值但保持“降级审计”，不得作为正式投注结论 |

### 理论亚洲盘计算修正（2026-07-14 01:27）

- 已废止风险引擎内部的 `(主胜概率-0.5)×4` 线性近似；
- 使用显式 `lambda_home`、`lambda_away`、`rho` 生成Poisson+Dixon-Coles比分矩阵；
- 遍历 `-5.0` 至 `+5.0` 的四分之一球盘口；
- 对每条盘口分别计算全赢、赢半、走盘、输半和全输；
- 选择主队公平十进制赔率最接近2.00的盘口作为理论盘；
- 缺少模型λ时返回不可计算，不再用固定欧赔区间或单一主胜概率补值；
- 水位差只允许在开盘与即时盘口相同的公司内计算。
