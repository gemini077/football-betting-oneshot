# 自动赛后验证与复盘

## 调度原则

- 每日定时任务只更新体彩赛程和统一主页，不运行赛后扫描。
- 用户选择场次并生成赛前报告时，系统同时记录开赛时间和唯一验赛时间。
- 普通比赛：开赛后 135 分钟执行一次。
- 单回合淘汰赛：开赛后 195 分钟执行一次，为加时和点球预留时间；模型结算仍使用 90 分钟含伤停赛果。
- 到点后若结果源尚未更新，只在 45 分钟后重试一次。第二次仍无法确认则标为阻塞并停止，不继续轮询。

## GitHub Actions 实现

- `scripts/postmatch_schedule.py` 为单场报告创建调度记录。
- `scripts/sync_postmatch_workflow.py` 把尚未执行的具体时间写入 `.github/workflows/postmatch-once.yml`。
- `scripts/postmatch_result.py` 只处理已经到期的场次；确认结果后删除对应未来触发，或将它替换成唯一一次延迟重试。
- 每次报告生成会同步上述工作流；不存在固定每 30 分钟或每小时扫描。

## 赛果识别

- 优先使用报告中的 500.com `shuju_id`。
- 报告没有该编号时，按业务日、主队和客队从比赛列表精确解析编号。
- 验证结果单独写入 `data/postmatch_automation/results/`，保留来源、验证时间和 90 分钟结算口径。
- 无法确认时不猜比分，也不生成伪复盘。

## 投注边界

- 赛果验证不会修改余额或创建注单。
- 未经用户明确说“锁单/已下单”，任何候选方案都不会写成已锁单或已投注。
- 只有真实存在的锁单，才允许在后续复盘中计算盈亏。
