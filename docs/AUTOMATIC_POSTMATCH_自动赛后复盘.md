# 自动赛后复盘与网页台账

## 目标

在无需逐场对话的情况下，定时扫描已经生成的赛前模型报告。比赛结束后核验90分钟赛果，执行固定9层复盘，更新Excel并重新生成网页台账。

## 自动队列

- 来源：`data/analysis_reports/**/*.json`。
- 只有包含完整主胜/平局/客胜概率的模型报告才进入队列。
- 同一场比赛只保留最新赛前报告。
- 普通比赛在开赛后135分钟安排一次复盘唤醒。
- 单回合淘汰赛在开赛后195分钟安排一次复盘唤醒，为加时和点球留出时间；复盘仍以90分钟赛果结算。
- 不允许固定每30分钟或每小时唤醒模型扫描。赛前报告生成时，同时用`scripts/postmatch_schedule.py`登记该场唯一一次唤醒时间。
- 若到点仍为延期、中断或未完赛，只允许45分钟后重试一次；之后保留未决状态。
- 已复盘对阵按比赛标识和对阵幂等跳过。
- 数据审计页、纯赔率报告和缺少开赛时间的报告不会自动写入复盘表。

## 赛果核验

- 统一使用90分钟含伤停时间赛果，不含加时和点球。
- 每场至少需要两个可追溯来源；来源冲突时暂停该场，不猜测结果。
- 加时与点球单独记录。

## Excel更新

- 最新工作簿由`05_RUNTIME_STATE.json`的`latest_review_workbook`指定。
- 新比赛使用下一个MatchID，并按记录ID递增追加。
- 锁单字段保持冻结；只有已存在且已明确锁单的注单才允许补结算字段。
- 未锁单的赛前候选永远不会自动写成已锁单或已投注。

## 网页台账

- 生成器：`scripts/postmatch_dashboard.py`。
- 固定入口：`data/postmatch_dashboard/latest.html`。
- 历史快照：`data/postmatch_dashboard/YYYYMMDD_HHMMSS/index.html`。
- 网页提供总览、赛前信号、结算明细、盘口时间轴和根因修正五个视图。

## 失败隔离

- 单场赛果无法双源核验时只保留在队列，不影响其他比赛。
- 复盘JSON未通过`schemas/postmatch_review.schema.json`时不更新工作簿。
- 工作簿更新后必须检查记录ID顺序和公式错误，再替换运行时的最新工作簿指针。
- 无比赛到期时不会唤醒模型，因此不会产生周期性空跑token消耗。
