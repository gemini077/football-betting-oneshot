# 统一赛事工作台

稳定入口：`data/match_workspace/latest.html`

每日任务虽然会保留带日期时间的历史快照，但只将上述 `latest.html` 作为用户入口；赛程 JSON 和历史快照都不应作为另一个网页发送给用户。

## 固定流程

1. 每天 10:00 只刷新中国体彩竞彩足球赛程。
2. 用户在工作台点击“加入待分析”，写入 `data/match_workspace/selected_matches.json`。
3. 选择只代表待分析，不会自动运行模型；用户明确要求分析后才抓深层数据并生成赛前报告。
4. 报告生成后，主维度的确认概率可独立发布给实时 EV 悬浮窗，不要求当时已经存在正 EV 候选。
5. 比赛结束后，单次赛后任务更新 Excel、复盘看板并重建统一工作台。

## 今日投注组合

- 保本层、中轴层、博上层是当天全部已分析场次的资金组合，只在统一工作台汇总，不属于任何单场报告。
- 单场报告只提供该场候选；工作台按票据层级聚合候选、已锁票据、跨场串联审核和同场重复暴露。
- 三层允许同时空仓。没有独立通过价格审核的候选时，不为了凑组合强行生成串关。
- 候选层级和金额不等于锁单；只有用户明确说“锁单/已下单”后才进入真实暴露。

## 赛程与复盘状态

- 已有90分钟赛果和复盘记录的比赛必须从未开赛列表移入已完赛列表。
- 赛后任务可以生成历史快照，但历史业务日不得覆盖固定 `latest.html`；完成复盘后应按当天日期重建统一主页。
- 已完赛行分别提供“赛后复盘”和“赛前分析”，复盘总表由卡片标题处单独进入。
- 单场复盘弹窗保留当前比赛上下文，在赛前分析、赛后复盘和复盘总表之间切换时不得丢失比赛。

## 安全边界

- 每日任务不执行自动分析、EV 筛选、锁单或下注。
- 悬浮窗只对报告精确绑定的主维度盘口赋值；切换到其他盘口时不复用错误概率。
- 页面选择、赛前报告和赛后复盘均不修改余额或暴露。
- 未收到用户明确的“锁单/已下单”，任何状态都保持未锁单。

## 手动命令

```powershell
python scripts\daily_schedule_workspace.py --no-cache
python scripts\match_workspace.py --date YYYY-MM-DD
```

每日刷新会同时请求当天与下一业务日已经开售的体彩赛程；仍然只更新赛程和总览，不自动运行分析。
